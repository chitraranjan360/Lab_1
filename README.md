1. What can JavaScript do that HTML/CSS cannot?
JavaScript can make pages interactive and dynamic — like responding to button clicks, validating forms, fetching data, and updating content without reloading the page.

2. Why prefer const by default?
Because it prevents you from accidentally changing a value. You only use let when you know the value will change.

3. == vs ===?

== checks value only → "5" == 5 is true
=== checks value and type → "5" === 5 is false

Always use === to avoid unexpected bugs.

4. Most useful console command?
console.log() — because it lets you see what's inside your variables at any point, making debugging much easier.

5. CSS hover vs JS hover?
CSS hover is more robust for simple styling because it's built into the browser, faster, and needs no code. JS hover is better when you need logic or dynamic behavior on hover, but it's overkill for just changing a color.
